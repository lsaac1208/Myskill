# Skills 文档模板使用指南

本目录包含两套标准化的 Skill 文档模板，用于创建高质量、一致的 Skill 文档。

## 📁 模板文件

### 1. `github-skill-template.md`
**用途**: 为 GitHub 仓库包装型 Skills 创建文档

**适用场景**:
- 包装现有的 GitHub 开源工具（如 zx, task, just 等）
- 工具本身有独立的 GitHub 仓库
- 需要追踪上游仓库的更新

**特点**:
- 包含 `github_url` 和 `github_hash` 字段
- 强调工具的原生功能和使用方法
- 提供详细的安装和配置指南
- 包含常见问题解答

### 2. `custom-skill-template.md`
**用途**: 为自定义开发的 Skills 创建文档

**适用场景**:
- 专门为 Claude Code 开发的功能
- 集成多个工具或服务的复合功能
- 特定领域的专业工具（如 docx, xlsx, pptx 等）

**特点**:
- 不包含 GitHub 追踪字段
- 强调 Skill 的使用场景和工作流
- 包含技术细节和最佳实践
- 提供完整的示例和注意事项

## 🚀 使用方法

### 创建新的 GitHub Skill

1. 复制模板：
```bash
cp /Users/wang/.claude/skills/.templates/github-skill-template.md \
   /Users/wang/.claude/skills/[new-skill-name]/SKILL.md
```

2. 填写 Frontmatter：
   - `name`: Skill 名称（小写，用连字符分隔）
   - `description`: 触发场景和关键词
   - `github_url`: 上游 GitHub 仓库 URL
   - `github_hash`: 当前对应的 commit hash
   - `version`: 版本号（建议从 0.1.0 开始）
   - `dependencies`: 依赖列表
   - `license`: 许可证类型

3. 填写文档内容：
   - 替换所有 `[...]` 占位符
   - 添加实际的代码示例
   - 补充常见问题和解决方案
   - 添加参考资料链接

4. 验证文档：
   - 确保没有遗漏的占位符
   - 检查代码示例是否可运行
   - 验证链接是否有效

### 创建新的自定义 Skill

1. 复制模板：
```bash
cp /Users/wang/.claude/skills/.templates/custom-skill-template.md \
   /Users/wang/.claude/skills/[new-skill-name]/SKILL.md
```

2. 填写 Frontmatter：
   - `name`: Skill 名称
   - `description`: 触发场景和关键词
   - `version`: 版本号（建议从 1.0.0 开始）
   - `entry_point`: 主入口脚本路径
   - `dependencies`: 依赖列表
   - `license`: 许可证类型

3. 填写文档内容：
   - 详细描述使用场景和工作流
   - 提供完整的代码示例
   - 说明技术实现细节
   - 添加最佳实践和注意事项

4. 验证文档：
   - 确保文档完整性
   - 测试所有示例代码
   - 检查依赖项是否正确

## 📋 文档标准

### 必需章节

所有 Skill 文档必须包含：

1. **Frontmatter** - 元数据
2. **适用场景** - 触发条件和关键词
3. **核心功能** - 主要功能列表
4. **使用方法** - 基本用法和示例
5. **安装** - 依赖安装和配置
6. **常见问题** - FAQ 和故障排除
7. **更新日志** - 版本历史

### 可选章节

根据需要添加：

- **高级特性** - 进阶功能
- **技术细节** - 实现原理
- **最佳实践** - 使用建议
- **性能优化** - 优化技巧
- **相关资源** - 扩展阅读

### 格式规范

1. **标题层级**:
   - H1 (`#`): Skill 名称
   - H2 (`##`): 主要章节
   - H3 (`###`): 子章节

2. **Emoji 使用**:
   - 🎯 适用场景
   - ✨ 核心功能
   - 🚀 使用方法
   - 📋 列表/表格
   - 🔧 安装配置
   - 📝 示例/日志
   - 🐛 问题/调试
   - 📖 文档/资源
   - 💡 提示/建议
   - ⚠️ 警告/注意

3. **代码块**:
   - 使用正确的语言标识符
   - 添加注释说明
   - 保持缩进一致

4. **链接**:
   - 使用描述性文本
   - 验证链接有效性
   - 优先使用官方文档

## 🔍 质量检查清单

创建或更新文档后，请检查：

- [ ] 所有占位符 `[...]` 已替换
- [ ] Frontmatter 字段完整
- [ ] 代码示例可运行
- [ ] 链接有效
- [ ] 无拼写错误
- [ ] 格式一致
- [ ] 示例充足
- [ ] FAQ 实用
- [ ] 版本号正确
- [ ] 更新日志完整

## 📊 版本管理

### 版本号规则

遵循语义化版本 (Semantic Versioning):

- **主版本号 (Major)**: 不兼容的 API 变更
- **次版本号 (Minor)**: 向后兼容的功能新增
- **修订号 (Patch)**: 向后兼容的问题修正

示例:
- `0.1.0` - 初始版本（GitHub Skills）
- `1.0.0` - 初始版本（自定义 Skills）
- `1.1.0` - 新增功能
- `1.1.1` - 修复 bug
- `2.0.0` - 重大变更

### 更新日志格式

```markdown
### v1.2.0 (2026-01-26)
- ✨ 新增 [功能描述]
- 🐛 修复 [问题描述]
- 📝 更新 [文档描述]
- ⚡ 优化 [性能描述]
- 🔧 调整 [配置描述]
```

## 💡 最佳实践

1. **保持简洁**: 文档应该简洁明了，避免冗余
2. **示例优先**: 提供实际可运行的代码示例
3. **用户视角**: 从用户角度编写，而非开发者视角
4. **持续更新**: 随着功能变化及时更新文档
5. **测试验证**: 确保所有示例和命令都经过测试
